# Production artifacts

This directory holds **production bundles**: frozen snapshots of the exact
fabrication artifacts for a specific board revision — the gerbers you actually
order, together with the PCB and outlines they were generated from.

Everything else generated by Ergogen (`outlines/`, `pcbs/`) and by KiBot
(root `gerbers/`) is a **live build product** and is intentionally *not* committed
(see `.gitignore`). `ergogen/config.yaml` is the single source of truth; run
`make build` (or download the CI build artifacts) to regenerate the current
design at any time.

## Layout

```
production/
  <order-id>/                 # e.g. 2026-08-01-jlcpcb
    gerbers/
      limbatus/               # base-board gerbers
      limbatus.zip            # the zip actually uploaded to the fab
      limbatus_autorouted/    # autorouted-board gerbers (if used)
      limbatus_autorouted.zip
    limbatus.kicad_pcb        # the exact PCB the gerbers came from
    outlines/                 # matching board/case outlines
    BOM.csv                   # optional: BOM as ordered
    SOURCE.md                 # provenance: source commit, date, fab, order id
```

A bundle is **immutable** once committed. It is guaranteed internally consistent
(gerbers ⇄ PCB ⇄ outlines all from one build), which is the property root-level
live outputs cannot promise as the design evolves.

## Cutting a new production bundle

Bundles are created and committed **by hand** — nothing in CI writes here.
When you are ready to place an order:

1. Regenerate from the commit you intend to fab: `make build`, then run the
   KiBot/autoroute steps to produce `gerbers/` (or pull them from the `Build`
   workflow's `kibot-default` artifact for that commit).
2. Copy the artifacts into `production/<order-id>/`:
   - `gerbers/`            → `production/<order-id>/gerbers/`
   - `pcbs/limbatus.kicad_pcb` → `production/<order-id>/limbatus.kicad_pcb`
   - `outlines/`          → `production/<order-id>/outlines/`
3. Write `production/<order-id>/SOURCE.md` recording the source commit, date,
   fab, and order id (copy an existing bundle's `SOURCE.md` as a template).
4. Commit the bundle and tag it: `git tag fab-<order-id> && git push --tags`.

Because `outlines/`, `pcbs/`, and root `gerbers/` are gitignored, use
`git add -f production/<order-id>` — the `production/` path itself is not
ignored, so a normal `git add production/<order-id>` works too.
